# AI起名站 — 自主赚钱系统设计文档

## 项目目标

开发一个AI自主运营的起名网站，实现以下指标：
- 日收入：100 RMB（约5单付费 × 19.9元）
- 用户介入：开发完成后，只做例外处理，日常零介入
- 核心价值：AI自动生产SEO内容引流 + 付费生成高质量起名结果

## 产品定义

### 三个功能入口

**入口1：宝宝取名（主力产品）**
- 输入：姓氏、性别（男/女/未知）、出生日期选填、风格偏好（诗经/楚辞/现代/英文名）
- 免费：生成10个名字 + 基本寓意（3-5字解释）
- 付费19.9元：生成50个名字 + 详细寓意（100-150字/名） + 八字五行分析 + 小名推荐 + 重名度评估

**入口2：公司/品牌起名**
- 输入：行业、关键词、风格（科技感/传统/简约/国际范）
- 免费：10个名字 + 基本解释
- 付费19.9元：50个名字 + 商标可注册性简要分析 + 域名可用性 + 品牌寓意拆解

**入口3：起名攻略文章（SEO流量池）**
- AI自动生成的长尾关键词文章
- 结构：知识科普 + 工具引导CTA

### 用户旅程

```
搜索"龙年男孩取名" → 进入文章页 → 看到工具入口
  → 免费试用生成10个 → 想要更多 → 付费19.9元
  → 获得完整结果 → 分享到微信/小红书（自然传播）
```

## 技术架构

### 技术栈

| 层 | 选型 | 理由 |
|------|------|------|
| 前端框架 | Next.js (App Router) | SSR对百度SEO友好，Serverless一条龙 |
| 部署 | Vercel (Hobby) | 免费、自动CI/CD、国内可访问 |
| AI模型 | DeepSeek (deepseek-chat) | 极便宜（百万token≈1元），中文起名质量足够 |
| 支付 | 面包多 | 个人即可接入，无需营业执照，微信+支付宝全支持 |
| 存储 | 初期无数据库 | MVP阶段支付回调记JSON文件，后续加SQLite/PostgreSQL |
| 样式 | Tailwind CSS | 快速开发，体积小 |
| 自动化脚本 | Node.js + Playwright | 知乎/小红书发布 + SEO内容自动生产 |

### 页面结构（8页）

1. `/` 首页 — 起名入口 + 热门名字 + SEO段落
2. `/baby-name` 宝宝起名页 — 表单 + 免费结果 + 付费解锁
3. `/company-name` 公司起名页
4. `/result/[id]` 起名结果页 — 付费后完整展示，可分享
5. `/articles` 文章列表页 — SEO文章目录
6. `/articles/[slug]` 文章详情页
7. `/pay/success` 支付成功回调页
8. `/about` 关于/联系页

### API路由

| 路由 | 功能 | 认证 |
|------|------|------|
| `/api/generate-free` | 免费生成10个名字 | 限流（IP级别，5次/小时） |
| `/api/generate-paid` | 付费生成50个名字+详细分析 | 支付凭证验证 |
| `/api/pay/create` | 创建支付订单 | 无 |
| `/api/pay/callback` | 支付回调 | 签名验证 |
| `/api/articles/generate` | AI自动生成SEO文章（内部触发） | 密钥认证 |
| `/api/sitemap` | 动态站点地图 | 无 |

## AI自主运营系统

### 流水线1：SEO内容生产（日更10篇）

```
Cron触发（每天9:00）
  → 抓取百度下拉词 + 知乎热搜起名词
  → 筛选低竞争高搜索量词（百度搜索结果少，月搜索量>100）
  → DeepSeek生成1500字原创文章
  → 自动发布到 /articles/[slug]
  → 更新sitemap提交百度收录
```

**文章类型：**
- "2026年X姓男孩名字大全"
- "龙年宝宝取名禁忌与宜用字"
- "诗经取名的100个好名字"
- "公司起名避坑指南：这5类名字注册不了商标"

**每篇文章结构：**
1. 引入（搜索意图满足）
2. 知识/干货内容（800-1000字）
3. 实操建议
4. CTA：用AI起名工具免费试试

### 流水线2：知乎/小红书内容分发

**知乎自动化流程：**
```
每4小时抓取"起名"相关新问题
  → 筛选高质量问题（关注数>5，回答数<10）
  → DeepSeek撰写走心回答（500-1000字，真实有用）
  → 文末自然引导："如果想多看看，这里有个AI工具免费生成10个名字 [链接]"
  → Playwright自动发布
```

**小红书自动化流程：**
```
每周3-5篇笔记
  → 内容形式：截图式（AI生成的名字列表截图）+ 走心文案
  → 标题："帮粉丝起的10个仙气古风名字" "做母婴博主3年整理的取名秘籍"
  → Playwright自动发布
```

### 流水线3：QQ群冷启动（OpenClaw Bot）

```
Bot 24小时在线
  → 搜索"宝妈群""孕妈群""新手爸妈""取名"等关键词加群
  → 群内自然参与，不主动广告
  → 检测到起名相关讨论时，主动帮忙："我试一下哈，给你起了几个名字：婉清、若兰..."
  → 被追问时引导到网站
```

Bot对话策略预设，OpenClaw执行。策略包括：
- 识别起名意图的关键词匹配
- 不同场景的回复模板
- 活跃度管理策略（不频繁说话，保持自然感）
- 群规检测（避免被踢）

## 开发阶段

### Phase 1：网站核心（3-4天）
- Next.js项目初始化 + Tailwind
- 首页 + 宝宝起名页 + 公司起名页
- `/api/generate-free` 接入DeepSeek
- Vercel部署 + 域名配置

### Phase 2：支付闭环（1-2天）
- 面包多支付接入
- `/api/generate-paid` 付费生成
- 付费结果页 + 支付成功页

### Phase 3：SEO基础设施（1-2天）
- 文章系统 + 动态sitemap
- SEO文章自动生成脚本
- 百度收录提交

### Phase 4：自动化分发（2-3天）
- 知乎发布脚本
- 小红书发布脚本
- 定时任务配置（Cron/GitHub Actions）

### Phase 5：QQ Bot接入（1-2天）
- OpenClaw配置 + QQ Bot策略注入
- 群搜索与加入逻辑
- 自动回复与引流策略

### Phase 6：监控与优化（1-2天）
- 简单管理后台（查看订单+收入+流量）
- 异常告警（支付异常、生成失败）
- 性能优化

**总预估：9-15天**（与你初期参与的1-2周一致）

| 风险 | 概率 | 缓解措施 |
